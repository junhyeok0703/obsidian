# 1. 실험 배경 및 가설

## 배경

- 이 실험을 진행하는 직간접적인 배경은 무엇인가요?
- 모르는 사람들도 이 문서를 봤을 때 실험의 출발점을 이해할 수 있도록 작성해 주세요.

## 문제 정의

- 어떤 문제를 해결하기 위한 실험인가요?
- 실험을 해야 하는 이유를 팀 구성원들이 공감할 수 있도록 작성해야 합니다.
- 문제 정의가 도출된 참고 자료(정량 분석 결과, 사용자 인터뷰 등)가 있다면 함께 적어도 좋습니다.
- 모든 문제를 실험으로만 해결하는 것이 정답은 아닙니다. 문제 정의를 정리하는 과정에서 이 문제를 실험으로 해결하는 것이 적절한지 다시 한 번 점검할 수 있습니다. 만약 팀원들이 실험의 필요성에 공감하지 못한다면, 문제 정의를 더 명확하게 다듬을 수 있는 기회입니다.

## 목표

- 이 실험을 통해서 달성하고자 하는 정량적/정성적 목표가 무엇인가요?
- 종종 실험을 설계하다 보면 원래 달성하고자 하는 목표는 잊은 채, 하나의 실험으로 여러 가지 목표를 다 달성하고자 하는 경우가 있습니다.
- 이 실험을 통해서 얻고자 하는 목표가 무엇인지 딱 하나를 정의해 주세요.

## 가설

- 가설도 마찬가지입니다. 하나의 실험으로 여러 가설을 검증하려다 보면 본래의 목적을 잃을 수 있습니다.
- 이 실험을 통해서 검증하고자 하는 가설이 무엇인지를 명확하게 정의해 주세요.
- 또한, 데이터로 검증 가능한 형태로 작성해야 합니다.
    - 예: x를 하면 y 지표가 증가할 것이다.

# 2. 실험 계획

## 실험 대상

- 실험의 대상이 되는 사용자를 정의합니다.
    - 특정 유입 채널, 특정 지역, 특정 운영 체제, 특정 성연령 등 제약 조건이 있다면 작성해 주세요.
    - 예: 해당 지면에 방문하는 전체 사용자 또는 광고 채널로 유입된 사용자

## 그룹 분배 방식

- 실험 그룹을 어떻게 배분할 예정인가요?
- 무작위로 배분할 것인지, 일정 비율로 나눌 것인지 명시해 주세요.
- 전체 사용자를 모두 실험 대상으로 할 수도 있지만, 만약 리스크가 예상되는 실험이라면 실험 그룹을 전체의 10% 정도로 차츰 늘려가는 방법도 있어요.

## 적정 실험 샘플 사이즈

- 적절한 샘플 사이즈는 얼마인가요?
- 샘플 사이즈 계산기를 참고해서 최소한 어느 정도의 샘플 사이즈를 확보하면 좋을지를 논의해 보세요.
    - 참고: [샘플 사이즈 계산기](https://www.abtasty.com/sample-size-calculator/)

## 실험 기간

- 실험 시작일과 종료일이 언제인가요?
- 보통 평일과 공휴일의 지표 양상이 다르기 때문에 최소 1주일 이상은 실험 기간을 잡는 것이 좋습니다.
- 단, A/B 테스트는 상황에 따라 조기 종료 및 연장될 수 있습니다.

## 실험 중간 결과 공유 일정

- 언제 실험의 중간 결과를 논의할 예정인가요?
- 가능한 경우, 실험 중간 결과에 대해서 논의하는 일정을 미리 잡아 두면 좋습니다.

## 그룹별 시안

# 실제 작성 예시

앞선 레슨들에서 살펴본 `코드비앤비` 예약 버튼 사례를 기반으로 한 실험 설계서 작성 예시를 참고해 보세요.

## 1. 실험 배경 및 가설

### 배경

- 전사적으로 20XX년도 KPI를 달성하기 위해서 예약률 증대를 위한 프로젝트 필요

### 문제 정의

- 데이터 분석 결과, 예약 버튼 클릭률이 약 10% 내외 → 클릭률을 더 높일 필요성이 있음
- UX 리서치 결과, 일부 PC 디바이스에서는 "예약 버튼이 하단에 있어서 눈에 잘 띄지 않는다"는 사용자 코멘트가 있었음 → 예약 버튼이 빠르게 인지되지 않음

### 목표

- 예약 버튼의 클릭률 상승

### 가설

- 예약 버튼의 위치를 가격 정보보다 위에 노출하면, 사용자가 더 빠르게 인지하고 클릭률이 높아질 것이다.
